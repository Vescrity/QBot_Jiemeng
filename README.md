# QBot_Jiemeng
借助go-cqhttp，使用C++开发的QQ聊天机器人程序

- 最近的版本中使用MATLAB来实现了一些特殊功能
- 本说明适用于v20.11
- 本说明用于介绍桔梦基本功能，组成，并帮助您快速上手。
- 具体的详细说明请参照进阶说明。

## 简介
桔梦(QBot_Jiemeng)是一个借助[go-cqhttp](https://github.com/Mrs4s/go-cqhttp)来达到QQ聊天功能的机器人程序。使用者可通过桔梦来实现检测关键词并做出特定回复的功能。词库可自定义。

## 系统需求

建议使用Windows 64位系统。

## 组成

桔梦的工作需要以下文件：
- 主体程序(默认名称Jiemeng.exe)
	- 负责进行消息处理等任务
- go-cqhttp及其相关文件
	- 用于接收、发送消息
- 启动器start.exe
	- 启动程序
- symbol程序
	- 作为标记表明程序正常运行
- 相关配置文件
- 词库文件
- 插件及其附属文件(非必须)
- 附加程序(非必需)


## 使用说明

### 使用前：

#### 下载

1. 下载新版桔梦release
2. 按release中说明下载[go-cqhttp](https://github.com/Mrs4s/go-cqhttp)

#### 配置

1. 打开go-cq程序以生成.bat文件

2. 配置go-cqhttp相关配置[config.yml]
	- a.账号密码
	- b.监听端口与反向HTTP端口
		- 监听端口设置为任意空闲端口【参考：5702】
		- 反向HTTP POST：
			- url: 'http://127.0.0.1:任意空闲端口（不可为监听端口）/'
			- 【参考：5703】
	- c.完成配置后请运行go-cqhttp.bat并按提示完成登陆

2. 配置Jiemeng相关配置[config.cfg]
	- (可参考预设文件)
	- 以下非必填内容可不出现在配置文件中，但一旦出现该字段则必须填写。
	- 配置文件中各字段前后顺序需保持与下方顺序一致。
	- 未来的计划：更改为json文件。

| 字段 | 说明 | 默认值 |
|-|-|-|
|Self_ID|bot qq号|0|
|INPORT|反向HTTP中的端口号|必填|
|OUTPORT|监听端口|必填|
|sleep_time|每次做出应答后的冷却时间|0|
|delay_time|发送多条消息时的间隔时间|0|
|TITLE|程序标题|桔梦+Self_ID|
|ANS_FILENAME|应答库文件名|Ans.ans|
|NOTE_FILENAME|笔记文件名|note.txt|
|IF_HAVE_MATLAB|是否安装MATLAB(是=1，否=0)|0|
|AL_TOKEN|alapi.cn 获取到的TOKEN|0|
|BAIDU_APP_ID|百度翻译API|0|
|BAIDU_APP_SECRET|百度翻译API|0|
|OPEN_AI_SECRET|使用ChatGPT功能需要|0|
|OPEN_AI_MODEL|ChatGPT模型名称|0|
|AI_PATH|stabble-diffusion-webui图片输出文件夹|0|
|SYMBOL_NAME|symbol程序的文件名|必填|

4. 配置start相关配置[start.cfg]
	- 第一行: Jiemeng主程序的文件名
	- 第二行: go-cqhttp的exe文件名
	- 第三行：是否显示来自go-cqhttp的信息(show/hide)
	- 第四行: symbol程序的文件名
5. Bot权限配置
	- a. [black_list.lst]
		- 全局黑名单，Bot不会对名单内的qq账号的消息做出响应
	- b. [white_list.lst]
		- 群聊白名单，Bot仅会对名单内的群号内的消息做出响应
	- c. [pri_white_list.lst]
		- 私聊黑/白名单。
		- 第一行black表示黑名单，white表示白名单
		- 黑白名单配置仅对私聊生效。
	- d. [priv_list.lst]
		- 管理权限列表。
			- 名单内的账号拥有对Bot的管理级权限。
	- e. [broad_pri.lst]
		- 私聊广播对象列表。
			- 使用广播指令时会对名单内的帐号发送广播消息。
#### 应答库文件


##### 结构

###### 应答组

应答库由若干个应答组组成。

每个应答组包含关键词部分，回复部分和优先级这三部分。
- 关键词部分由若干组关键词组成，每组关键词可含有若干个关键词
- 回复部分则是若干组回复组成，每组回复可包含若干条回复


##### 逻辑

- 首先所有的应答组依照优先级排序，随后每收到消息，程序从排名靠前的应答组开始逐一检查各应答组条件是否满足。满足则进行应答，不满足则检索下一组。
- 对于一个应答组，满足条件需要满足其关键词部分的条件
- 而关键词部分的条件则是满足任意一组关键词的条件。
- 对于一组关键词，则是需要满足收到的消息中，存在着这一组关键词中的所有关键词。
- 关键词组也可以设置成需要“精确匹配”，这样只有当消息内容【正好】是关键词时才会做出应答。所谓【正好】就是一字不多，一字不少。
- 而应答的逻辑则是随机从该应答组中的回复部分抽取一组回复，将回复逐条输出。
##### 格式
- 格式具体可参考示例文件。
- 关键字，文本信息等内容分别独立成行。
- Key_st
	- 这个关键字用来声明一组应答组开始。
	- 除此之外也可以使用【spid=qq号】，【sgid=群号】，【Priv_key】这几种关键字来声明。
	- 【spid=qq号】对这个应答组会增加修饰，表明该应答组仅当收到这个qq号的消息时才可能满足条件。
	- 【sgid=群号】同理，使得该应答组仅对该群生效。
	- 【Priv_key】仅有qq号写在了priv_list.lst文件中的账号可触发
- Key_st后一行是对关键词部分的声明。每一组关键词仅需要让每一个关键词单独占用一行。
- Key_0
	- 这个关键字表明一组关键词的声明结束。接下来可以选择继续声明新的一组关键词或结束关键词部分声明
	- 【Key_1】是这个关键字的特殊形式。它将会对这组关键词进行修饰，使得其条件为【精准匹配】。显然，当精准匹配开启时，关键词仅能为一条，否则将会是无效的。
- Key_ed
	- 这个关键字表明对关键词部分的声明的结束。接下来开始声明回复部分。
- Key_ed后一行是对第一组回复的声明。每一条回复占用一行。
- Ans_or
	- 这个关键字用来分隔不同组回复。
- Pri
	- 当声明完所有组回复后，使用【Pri】来结束回复部分的声明，并开始对优先级的声明。
	- 下一行为一个数字，表示优先级。并且结束这一应答组的声明。
- //end
	- 表明整个文件结束
- #开头的一行表明是注释，程序将会忽略这一行。

##### 文本内容常见替换
出于一些操作需求，一些特殊的字符串将会在输出前被替换成其他内容。部分如下：
- [\n]
	- 由于每条回复必须占据一行，这样无法实现一条多行的回复。在回复内写【[\n]】可以在输出时达到换行的效果。
- [poke]
	- 单独写成一条回复，表明发送一条戳一戳（头像双击）
- [name]
	- 替换为发信息者的群名片/昵称
- [rand:数值]
	- 替换为0-【数值-1】的一个随机数
- 更多特殊代码详见进阶说明。

### 开始使用：
- 请确保已完成所有配置
- 将Jiemeng的全部文件内容放到和go-cqhttp同一文件夹下，运行start.exe即可
