## 关键词部分

在关键词部分也预先定义了一些特殊的代码使得其可以支持特定功能。需要注意这些关键词均需要修饰为【精确匹配】

| 代码 | 功能 |
|--|--|
|[default]| 当在收到没有任何关键词满足条件时的私聊消息时将会触发该条关键词 |
|[poke]|被戳（头像双击）时会触发该关键词|
|[CQ:at,qq=...]|表示at某人|
|[NO image]|当试图发送[CQ:image,file=]时触发，一般在搜图出现故障时会出现该问题|


## 管理命令


一些只有有权限人员可进行的进阶操作。
 - .resend
	 - 消息中发送【.resend内容】使bot认为在当前聊天中收到了该内容。同样也可以实现对需要[resend:...]的指令的触发。
- .repost
	- 消息中发送【.resend内容】使bot在当前聊天中发送该内容。
- .type_resend/.type_repost
	- 消息中发送【.type_resend#群号#qq号#内容】使bot认为在该群中收到了来自qq号的内容。群号参数填写长度为1~2的字符视为收到私聊。
	- 消息中发送【.type_repost#群号#qq号#内容】使bot在指定群聊中发送指定内容。群号参数填写长度为1~2的字符视为发送私聊。




## 进程保护start.exe

由于旧版本的桔梦具有较大的不足经常出现闪退现象，所以设计了这个程序。每5秒检查一次桔梦程序是否在运行，如未运行则启动。虽然当前版本稳定性已足够高，不过依然保留了这个程序，用于实现桔梦程序的重启功能。当桔梦收到[Restart]指令之后，在再次被触发任意消息后将会结束桔梦进程。随后等到start.exe下次检查的时候便会重新启动桔梦，同时可以实现对所有词库文件，黑白名单等配置的重新读取。请务必把start.exe和桔梦的程序与全部配置文件放置于同一文件夹内。

- start.cfg
	- 由于start.exe是通过检查特定名称的进程是否存在来进行“进程保护”的，所以当需要实现一台电脑挂多个账号的时候，就需要区别开每个账号的桔梦主程序名称。为了让start.exe知道这个账号的主程序名称，就需要一个配置文件来对程序做出说明。
	- 所以如果要使用start.exe，请在start.cfg中填写你**本账号文件夹内**的桔梦主程序名称。
	- 如对说明感到不够清楚请查看示例配置文件。

请注意，关闭start.exe请在界面中按Ctrl+C，如果误操作使用了‘x’进行关闭，请在任务管理器中结束start.exe的进程。

## 高级使用方法

桔梦的代码中提供了一套相对较为易用的信息发送的函数。你可以利用这些函数编写程序来进行比如批量的消息发送功能。
- 示例1
	- 将文本文件中的每行内容分条输出到特定群聊中。文件以//end结尾
	- ~~才不是懒得写EOF什么的呢！~~

示例如下：

   ```C++

#include<time.h>
#include<stdlib.h>
#include<windows.h>
#include"output.h"
#include"opstring.h"
#include"File_Read.h"
#include<stdio.h>
char bufrd[1024];
int main(){
	  system("chcp 65001");
	  Msg_type type;
	  get_copy(0,9,"123456789",type.grp_id);		//设定目标群号
	  type.ifgrp=1;						//使消息类型定义为群聊
	  OUTPORT=5706;						//发送消息的账号的端口，确定当前发消息账号
	  Self_ID=1;
	  int slptm=3300;					//冷却时间
	  freopen("readin.txt","r",stdin);			//重定向输入至文件
	  for(;;){
		    gets(bufrd);				//读取文件一行内容
		    if(strlen(bufrd)==0)continue;		//空白行忽略
		    if(bufrd[0]=='/')return 0;			//’/‘开头标志文件结束，停止读取
		    output(type,bufrd);				//将字符串bufrd发送
		    Sleep(slptm);                         	//冷却
	  }
}


```
